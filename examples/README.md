# examples/

This directory contains **reference inputs and filled examples** for working with the DevOS runtime.

## Relation to DevOS Architecture

Examples provide concrete, non-normative illustrations of how to populate the mandatory and optional project input files that DevOS requires before a workflow can start. They are reference material only — not part of the runtime or framework.

## Contents

| Directory | Purpose |
| --- | --- |
| `templates/mandatory/` | Empty templates for all mandatory project inputs (copy to `.devOS/project_inputs/` and fill in) |
| `templates/optional/` | Empty templates for optional quality-enhancing inputs |
| `filled/` | Filled example files illustrating correct structure and content |
| `filled/run_example/` | Example of a completed run directory with artifacts |

## What Belongs Here

- Input file templates (`.md`, `.yaml` stubs)
- Filled example inputs demonstrating correct structure
- Example run artifacts for reference

Do not place normative framework definitions, runtime code, or project-specific domain content here.
